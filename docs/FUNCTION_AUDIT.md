# 功能完成度审计

审计时间：2026-06-20

结论：当前实现已按“客户侧 ERP + 独立 OperationsPlatform”重新收口，但不能表述为 ERP 全功能已完整完成。README 中的 ERP 能力是目标业务面；当前代码达到“首版可运行闭环 + 部分关键操作入口”的状态，仍有若干深业务页面、报表钻取、异常处理和单据细节需要继续补深。

## 已修正边界

| 范围 | 状态 | 证据 |
| --- | --- | --- |
| ERP 前端默认入口 | 已修正 | `ERP/frontend/src/App.tsx` 默认进入“ERP 工作台”，不再导入或渲染 `ProductOpsView` |
| ERP 前端产品运营页面 | 已移除 | `ERP/frontend/src/views/ProductOpsView.tsx` 已删除 |
| ERP 前端 API 客户端 | 已修正 | `ERP/frontend/src/services/api.ts` 不再保留 `/product-ops/*` 调用方法 |
| ERP 前端类型 | 已修正 | `ERP/frontend/src/services/types.ts` 已移除运营台专用类型 |
| ERP API 身份 | 已修正 | `/api/` 返回 `common-build-materials-erp` |

| 仓库边界文档 | 已修正 | 根 README、交付说明、部署说明均明确 `OperationsPlatform/` 承接内部运营能力 |

| 范围 | 状态 | 说明 |
| --- | --- | --- |
| ERP 工作台 | 首版闭环增强 | 前端默认展示订单、生产、调度、地磅、签收、对账、采购、财务、审批和经营指标总览，并将基础资料分散到销售、履约、采购库存等业务模块；补充销售订单弹窗提交、采购入库、对账确认、收款/付款计划和审批处理入口 |
| 实验室管理 | 已实现首版 | 覆盖配比版本链、试配、审批生效/停用、生产试块、原料检验、样品试验复核、仪器校准约束、质量异常闭环、站点权限裁剪、审计日志、SSE 事件和前端工作台 |
| 基础资料 | 首版 CRUD | 客户/项目并入销售业务，产品/站点/司机/车辆并入履约中心，物料/库存并入采购库存；各台账页使用通用分页表格，提供查询、弹窗新增、弹窗编辑和删除，删除会校验业务引用；承运商、仓库、筒仓等仍需继续补齐独立维护页 |
| 销售到收款 | 首版闭环 | 覆盖订单、签收、对账、应收、收款和付款计划；合同、开票、红字、催收等已有接口但前端深操作仍需继续补齐 |
| 生产到发货 | 已实现首版 | 覆盖生产计划、任务、批次、调度、地磅、票据、车辆轨迹和工地签收 |
| 采购到付款 | 首版闭环 | 覆盖采购申请、采购订单、入库、库存流水、应付生成；供应商对账和付款的前端深操作仍需补齐 |
| 系统底座 | 首版可运行 | 覆盖授权校验、插件、审批流、数据字典、备份恢复、API Gateway、SSE/Redis/RabbitMQ 事件；系统配置和运维类页面仍需分模块补深 |

## 下一步建议

1. 继续补齐 ERP 客户侧深业务页面，而不是把产品运营能力挂回 ERP。
2. 将仍留在 `ERP/backend/internal/appliance/` 的历史产品运营实现逐步迁移到 `OperationsPlatform/` 或删除，避免后续误用。
3. 为销售、生产、调度、地磅、签收、结算、采购、财务、报表和系统模块补齐完整 CRUD 对账表；当前这些模块多数是“列表 + 新增/状态动作”，不能视为每个模块都有完整增删改查。
4. 为合同、开票/红字、催收、供应商对账/付款、地磅作废复核、报表钻取和系统配置补前端闭环测试。
5. OperationsPlatform 后续补深授权续费、异常告警、客户端/服务端更新包和端内 updater 执行链路。
