# 交付说明

当前交付的是独立的产品运营平台骨架，定位为厂商内部使用，不是客户侧 ERP，也不是 SaaS 多租户系统。

## 当前交付范围

| 范围 | 状态 |
| --- | --- |
| 独立项目目录 | 已完成，目录为 `CommonBuildMaterialsOperationsPlatform/` |
| 后端服务 | 已完成，Go 标准库 HTTP API |
| 前端控制台 | 已完成，静态 HTML/CSS/JS，可由后端托管 |
| 客户部署台账 | 已完成，按客户私有化部署实例维护 |
| 授权续费登记 | 已完成，续费后更新客户授权状态并生成审计日志 |
| 系统异常告警 | 已完成，支持打开、确认、关闭 |
| 客户端/服务端更新包 | 已完成，支持创建、发布、分配 |
| 本地数据持久化 | 已完成，JSON 文件存储 |
| GPS 转发器融合 | 已完成，作为同项目后端命令和 Docker 服务交付 |
| 自动采集客户心跳 | 未接入，后续由 ERP 客户端/服务端上报 |
| 更新包文件仓库 | 未接入，后续可接 MinIO、OSS 或内网制品库 |
| 登录权限 | 未接入，后续可接内部 SSO/OIDC |

## 与 ERP 的关系

ERP 仍然是 `CommonBuildMaterialsPlatform/`，部署在客户自己的服务器上，保留客户侧业务概念：站点、生产线、地磅、订单、调度、库存和财务。

运营平台只管理我方视角的产品交付对象：客户部署、授权、异常、版本和更新包。

## 后续建议

1. 增加 ERP 服务端心跳上报接口，自动刷新客户部署健康状态和服务端版本。
2. 增加桌面客户端更新检查接口，结合更新包分配记录控制灰度。
3. 接入内部 SSO，按产品运营、客户成功、交付运维区分操作权限。
4. 接入真实制品库，更新包记录只保存元数据和下载地址，不直接存二进制。
5. 将 GPS 转发器运行指标接入运营平台告警列表。
